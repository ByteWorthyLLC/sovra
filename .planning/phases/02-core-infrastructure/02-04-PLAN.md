<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| URL params -> Auth pages | Error messages from callbacks displayed to user |
| URL slug -> Tenant layout | Tenant slug from URL used for DB lookup |
| URL token -> Invite page | Invitation token from URL used for acceptance |
| Browser form -> Server action | User input (email, password, tenant name) crosses trust boundary |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-12 | Spoofing | Login form | mitigate | Zod validates email format and password min length; Supabase handles rate limiting and brute-force protection |
| T-02-13 | Information Disclosure | Error display | mitigate | Error messages from Supabase are user-friendly; no stack traces or internal details exposed in UI |
| T-02-14 | Tampering | Tenant slug in URL | mitigate | Layout validates slug against DB; notFound() for invalid slugs; RLS prevents cross-tenant data access |
| T-02-15 | Elevation of Privilege | Invite form | mitigate | usePermission('tenant:invite') gates the form; role options come from DB not user input; RBAC checked server-side in createInvitation |
</threat_model>
